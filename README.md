# 📌 クリニック共有付箋ボード

院内ネットワーク（LAN）に繋がっているPC・**タブレット・スマートフォン**なら、どの端末のブラウザからでも使える共有付箋（ふせん）システムです。
サーバーとなるPCを1台決めて起動するだけで、他の端末はブラウザでアドレスを開くだけで使えます。
**インストールが必要なのはサーバーPCの Node.js だけ**で、データベースや追加ソフトは不要です。

## できること

| 機能 | 説明 |
|---|---|
| 付箋の追加・編集・削除 | ブラウザから誰でも簡単に操作できます |
| 種類（色分け） | 🚨 **緊急**（赤・点滅）／ ⚠ **要注意**（オレンジ）／ **連絡**（青）／ **通常**（黄） |
| 表示 ON / OFF | 付箋ごとに「非表示」にできます。非表示の付箋は「非表示の付箋も見る」をONにした人だけに表示されます（下書きや、一旦隠したい連絡に） |
| 宛先 | 全員／受付／看護師／医師／事務 を指定でき、タブで絞り込めます |
| 完了チェック | 対応が終わった付箋は ✅ で「完了」欄へ。あとから戻すことも削除もできます |
| 🔔 ポップアップ通知 | 付箋を「ポップアップで通知する」にすると、対象端末の画面中央に**通知音付きで、確認ボタンを押すまで消えない通知**が表示されます |
| 端末の指定 | ポップアップの宛先は「すべての端末」または**特定の端末1台**（例: 受付1 だけ）を選べます |
| タブレット対応 | iPad などのタブレット・スマートフォンのブラウザでもそのまま使えます（タッチ操作向けにボタンを大きく調整済み） |
| 緊急件数の表示 | 緊急の付箋があると、画面上部とブラウザのタブに件数が点滅表示されます |
| 自動同期 | 5秒ごとに自動で最新の状態になります。どのPCで書いてもすぐ全PCに反映されます |
| 検索 | キーワードで付箋を絞り込めます |
| 名前の記録 | 画面右上に自分の名前を入れておくと、付箋に「誰が書いたか」が残ります |

## 使用イメージ（クリニックでの用途例）

- 「🚨 緊急」…… 本日の休診連絡、システム障害、至急の申し送り
- 「⚠ 要注意」…… 転倒リスクのある患者様の来院予定、アレルギー情報の注意喚起
- 「連絡」…… 業者の来訪予定、備品の発注済み連絡、会議の案内
- 「通常」…… ちょっとしたメモ、お願いごと

## セットアップ手順

### 1. サーバーにするPCを1台決める

常時つけているPC（受付のPCなど）がおすすめです。

### 2. Node.js をインストール（サーバーPCのみ・初回だけ）

1. https://nodejs.org/ja を開く
2. 「LTS（推奨版）」をダウンロードしてインストール（すべて「次へ」でOK）

### 3. このフォルダをサーバーPCに置いて起動

- **Windows の場合:** `start.bat` をダブルクリック
- **Mac / Linux の場合:** ターミナルでこのフォルダに移動して `node server.js`

起動すると黒い画面に、他のPCからアクセスするためのアドレスが表示されます。

```
  院内の他のPCからは、以下のいずれかのアドレスを
  ブラウザで開いてください:
    http://192.168.1.23:3000   ← 例
```

### 4. 各PC・タブレットのブラウザで開く

院内の各端末（受付・診察室・処置室のPC、タブレットなど）のブラウザ（Edge / Chrome / Safari など）で、
表示されたアドレス（例: `http://192.168.1.23:3000`）を開き、**ブックマーク（お気に入り）登録**しておくと便利です。

- タブレット・スマホは、院内のWi-Fi（同じネットワーク）に接続されている必要があります。
- iPad/iPhoneなら「ホーム画面に追加」、Androidなら「ホーム画面に追加」をしておくと、アプリのようにワンタップで開けます。

### 5. 各端末に「端末名」を付ける

画面右上の「端末名」欄に、その端末の名前（例: `受付1`、`診察室`、`処置室タブレット`）を入力してください。

- 端末名を付けると、その端末が自動でサーバーに登録され、**ポップアップ通知の宛先として選べる**ようになります。
- 端末名は各端末のブラウザに記憶されるので、入力は初回の1回だけでOKです。

## 🔔 ポップアップ通知の使い方

1. 付箋の作成（または編集）画面で「🔔 ポップアップで通知する」にチェック
2. 宛先を「すべての端末」または特定の端末（例: `受付1`）から選択（現在接続中の端末には「（接続中）」と表示されます）
3. 保存すると、対象の端末の画面中央に**通知音付きのポップアップ**が表示されます

- ポップアップは**「確認しました」ボタンを押すまで消えません**（見逃し防止）
- 確認したかどうかは端末ごとに記録されます。付箋の内容を編集すると、再度ポップアップが表示されます
- 未確認の通知が複数ある場合は、確認するたびに次の通知が順番に表示されます
- 使用例:
  - 診察室 → 受付だけに「次の患者様をご案内ください」
  - 全端末に「本日18時から院内ミーティングです」
  - ※ブラウザの仕様上、ページを開いてから一度も画面に触れていない端末では通知音が鳴らないことがあります（ポップアップ表示自体は出ます）

### Windows ファイアウォールについて

初回起動時に「アクセスを許可しますか？」というWindowsの確認画面が出たら、
**「プライベートネットワーク」にチェックを入れて「アクセスを許可する」** を押してください。
（これを許可しないと他のPCから繋がりません）

### PC起動時に自動でサーバーを立ち上げたい場合（任意）

`start.bat` のショートカットを作成し、スタートアップフォルダ
（`Win + R` → `shell:startup` で開くフォルダ）に入れておくと、PC起動時に自動で立ち上がります。

## データの保存場所とバックアップ

付箋のデータは `data/notes.json` という1つのファイルに保存されます。
このファイルをコピーしておけばバックアップになります。戻すときは同じ場所に置くだけです。

## ポート番号を変えたい場合

初期設定はポート `3000` です。他のシステムと重なる場合は、環境変数 `PORT` で変更できます。

```
set PORT=8080 && node server.js
```

## ⚠ 注意事項（必ずお読みください）

- このシステムは **院内LAN専用** です。インターネットに公開しないでください（ルーターのポート開放などは行わないでください）。
- ログイン機能はなく、院内LANに繋がっている人は誰でも読み書きできます。**運用ルール上、必要以上の患者様の個人情報（フルネーム・生年月日・病名など）は書かない**ことをおすすめします（例:「午後の予約 田中様」程度に留める）。
- 電子カルテの代わりになるものではありません。診療記録は必ず正規のシステムに記録してください。
